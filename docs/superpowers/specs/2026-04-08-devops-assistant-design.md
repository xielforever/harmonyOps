# 运维助手应用设计方案

## 1. 项目概览

**应用名称：** 运维助手 (HarmonyOps)
**应用类型：** 效率/工具类 (服务器运维管理)
**核心目标：** 提供一个移动端的服务器管理工具，支持状态监控、告警处理、SSH终端直连以及常用运维脚本执行。
**架构模式：** 纯原生 ArkUI (ArkTS) 开发 + 中心化后端 API (数据通过云端接口获取)。

## 2. 核心功能

1.  **服务器状态监控 (Monitor)：** 实时查看服务器的 CPU、内存、磁盘等核心指标。
2.  **告警通知与处理 (Alert)：** 接收和处理服务器/应用的异常告警。
3.  **SSH远程终端 (Terminal)：** 通过手机远程连接服务器执行命令。
4.  **常用运维脚本/操作 (Tools)：** 一键执行预设的常用运维脚本（如重启服务、发布版本等）。

## 3. UI/UX 设计规范 (参考 stitch_devops_mobile_prototype)

采用 **"Precision in Darkness" (高对比度暗色主题)**，打造专业开发者级别的 UI 体验。

### 3.1 颜色规范 (Colors)
*   **Primary (`#a78bfa`):** 柔和紫罗兰色 —— 用于交互元素、链接、焦点环。
*   **Background (`#09090b`):** 纯正近黑色 —— 应用主背景。
*   **Tertiary (`#34d399`):** 翠绿色 —— 用于成功状态、积极指示器、代码高亮。
*   **Surface scale:** 锌灰色系 (`#0c0c0f` → `#27272a`) —— 用于卡片、输入框等表面层级区分。
*   **Error (`#ef4444`):** 红色 —— 仅用于错误提示，不作装饰。

### 3.2 排版规范 (Typography)
*   **主字体:** Geist (或系统默认的无衬线黑体，保持现代、简洁、适合开发者)。
*   **主文本色:** `#fafafa`，确保高对比度。
*   **次文本色:** `#a1a1aa`。

### 3.3 视觉与组件规范
*   **卡片 (Cards):** 使用 surface 颜色背景，极细的边框 (`#27272a`) 进行分隔，无明显阴影，圆角 8px。
*   **按钮 (Buttons):**
    *   Primary: 纯紫罗兰色填充。
    *   Secondary: 透明背景 + 边框。
*   **分隔:** 优先使用边框而非阴影来区分层级，保持界面扁平和精确。

## 4. 页面规划

应用采用底部导航栏 (Tabs) 结构，分为以下四个主要页面：

### 4.1 概览页 (Dashboard)
*   **顶部:** 显示服务器总数和整体健康状态概览。
*   **内容区:** 卡片式展示各核心服务器的资源占用（CPU、内存、磁盘）图表或进度条。

### 4.2 终端页 (Console)
*   **列表区:** 展示可连接的服务器列表。
*   **交互区:** 点击服务器后进入全屏的 SSH 命令行交互界面（需适配暗色主题和等宽字体）。

### 4.3 告警页 (Alerts)
*   **列表区:** 按时间倒序排列的告警消息列表（卡片式）。
*   **状态:** 区分未处理（使用红色/醒目色提示）和已处理状态。
*   **操作:** 点击查看详情，支持确认、忽略等操作。

### 4.4 工具页 (Scripts)
*   **列表区:** 分类展示常用运维脚本（如：重启 Nginx、清理日志等）。
*   **操作:** 点击执行，并展示执行结果状态。

## 5. 核心技术栈

*   **UI 框架:** ArkUI (基于 ArkTS)。
*   **网络通信:** `@ohos.net.http` (RESTful API) 和 `@ohos.websockets` (用于 SSH 终端流式数据)。
*   **数据存储:** `@ohos.data.preferences` (存储用户配置、API Token 等)。
*   **组件封装:** 自定义图表组件、终端模拟器组件（适配 ArkUI）。
